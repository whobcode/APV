Pathfinding Algorithms

Learn about different pathfinding algorithms, their characteristics, and when to use them.

A* Search

Optimal Heuristic Weighted

The gold standard of pathfinding. A* uses a heuristic function to estimate distance to the goal, combining the actual cost from start with estimated cost to goal (f = g + h).

Time: O(E log V) Space: O(V) 
f(n) = g(n) + h(n)
where:
  g(n) = cost from start to n
  h(n) = heuristic estimate to goal

Dijkstra's Algorithm

Optimal Weighted

Explores all directions equally, guaranteeing the shortest path. Essentially A* with h(n) = 0. More thorough but slower than A* for single-destination searches.

Time: O(E log V) Space: O(V) 
// Always expand node with lowest g(n)
// No heuristic guidance
f(n) = g(n)

Breadth-First Search

Optimal* Unweighted

Explores nodes level by level. Guarantees shortest path only in unweighted graphs. Simple and efficient for uniform-cost scenarios.

Time: O(V + E) Space: O(V) 
// Use a queue (FIFO)
// Expand oldest nodes first
// All edges have cost 1

Depth-First Search

Not Optimal Unweighted

Explores as deep as possible before backtracking. Does NOT guarantee shortest path. Useful for maze solving and detecting connectivity.

Time: O(V + E) Space: O(V) 
// Use a stack (LIFO)
// Expand newest nodes first
// May find long paths

Greedy Best-First

Not Optimal Heuristic

Always moves toward the goal using only the heuristic. Very fast but does not guarantee shortest path. Can get stuck in dead ends.

Time: O(E log V) Space: O(V) 
// Only use heuristic
f(n) = h(n)
// Ignores actual path cost
